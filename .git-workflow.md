# Git Workflow Kullanım Kılavuzu

## Otomatik Workflow Script

Bundan sonraki tüm push işlemleri için aşağıdaki workflow kullanılacak:

### Kullanım

```bash
./git-workflow.sh "Commit mesajınız buraya"
```

### Workflow Adımları

1. **Branch Oluşturma**: Eğer main branch'indeyseniz, otomatik olarak yeni bir branch oluşturulur
   - Format: `update/YYYYMMDD-HHMMSS`
   - Örnek: `update/20250115-143022`

2. **Değişiklikleri Commit Etme**: Tüm değişiklikler otomatik olarak commit edilir

3. **Branch Push**: Yeni branch GitHub'a push edilir

4. **Main'e Merge**: Branch otomatik olarak main'e merge edilir

5. **Main Push**: Main branch push edilir ve Vercel otomatik deploy başlar

### Örnek Kullanım

```bash
# Basit commit mesajı
./git-workflow.sh "Fix: Build hataları düzeltildi"

# Detaylı commit mesajı
./git-workflow.sh "Feature: Yeni özellik eklendi - Kullanıcı profil sayfası"
```

### Manuel Kullanım (Script olmadan)

Eğer script kullanmak istemezseniz:

```bash
# 1. Yeni branch oluştur
git checkout -b update/feature-name

# 2. Değişiklikleri commit et
git add -A
git commit -m "Commit mesajı"

# 3. Branch'i push et
git push -u origin update/feature-name

# 4. Main'e geç ve merge et
git checkout main
git merge update/feature-name

# 5. Main'i push et (Vercel deploy başlar)
git push origin main
```

### Notlar

- Vercel, `main` branch'ine push yapıldığında otomatik olarak deploy başlatır
- Her push işlemi için yeni bir branch oluşturulur (geçmiş takibi için)
- Branch'ler GitHub'da kalır, silinmez (opsiyonel olarak temizlenebilir)
